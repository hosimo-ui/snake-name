# Snake Game 🐍

基于 **Pygame** 开发的经典贪吃蛇游戏，采用模块化架构，在传统玩法基础上融入了现代游戏设计理念。

## 🎮 快速开始

```bash
pip install pygame
python main.py
```

## ✨ 项目亮点

### 1. 清晰的模块化架构

项目按职责拆分为 5 个模块，各司其职、低耦合：

| 模块 | 职责 |
|------|------|
| `main.py` | 入口 + 游戏状态机循环 |
| `config.py` | 全局常量 / 枚举 / 颜色（消除魔术数字） |
| `entities.py` | 游戏实体类（Apple, Segment, Snake）+ AABB 碰撞检测 |
| `utils.py` | 工具函数（食物生成、最高分持久化、键盘输入） |
| `ui.py` | 界面渲染（HUD、菜单、暂停、结束画面） |

导入链为单向无环：`config → entities → utils → main` / `config → ui → main`，无循环依赖。

### 2. 完整的游戏状态机

采用四状态有限状态机管理游戏生命周期，状态切换逻辑清晰、职责分明：

- `MENU` — 展示操作指南，ESC 可直接退出
- `PLAYING` — 核心游戏循环，含碰撞检测与难度递增
- `PAUSED` — P 键随时暂停/恢复，半透明遮罩覆盖
- `GAME_OVER` — 结算界面，支持重玩（Y）或退出（N）

### 3. 持久化最高分系统

利用文件 I/O 实现本地持久化存储——最高分保存在 `high_score.txt`，游戏退出后依然保留。破纪录时显示 ★ NEW HIGH SCORE! 视觉反馈。文件读取失败时回退为 0，不阻塞启动。

### 4. 动态难度曲线

每获得 50 分帧率自动 +2 FPS（上限 45 FPS），蛇速渐进加快。新手友好、高手有挑战，且难度参数统一在 `config.py` 中可调。

### 5. 精准的碰撞检测与食物生成

- 采用 **AABB** 算法进行矩形碰撞判定，函数独立封装、职责单一
- 食物生成时调用 `Snake.getOccupiedPositions()` 获取蛇的**全部身体坐标集合**，确保食物不会出现在蛇身上（而非仅避开蛇头）
- 食物坐标对齐到 `SNAKE_SIZE` 网格，与蛇移动步长一致

### 6. 环绕边界机制

蛇穿越屏幕边界后从对侧出现（而非撞墙死亡），保留经典玩法气质，降低挫败感，让玩家专注于策略规划。

### 7. 丰富的视觉细节

| 元素 | 设计 |
|------|------|
| 蛇头 | 绿色方块 + 黑色眼睛（位置随方向变化） |
| 蛇身 | 黄色渐变着色（越靠尾越暗），直观反映身体长度 |
| 食物 | 橙色方块 + 黄色描边光晕 |
| HUD | 实时得分、最高分、存活时间 |

### 8. 防御性编程

- 禁止 180° 原地掉头（`setDirection()` 通过 opposite 字典判断）
- 食物生成避开蛇全身（`getOccupiedPositions()` 排除所有非哨兵段）
- 高分文件 I/O 异常静默回退，不阻塞游戏
- 自碰检测遍历所有非 NULL 段，不依赖位置假设
- 哨兵段（sentinel）机制：使用 `color="NULL"` 标记不可见占位段，实现自然的生长间隙

## 🎯 操作说明

| 按键 | 功能 |
|------|------|
| ↑ ↓ ← → | 控制移动方向 |
| P | 暂停 / 继续 |
| ESC | 退出（菜单中直接退出，游戏中进入结算界面） |
| Y / N | 结算界面：重新开始 / 退出 |

## 🗂 项目结构

```
snake-game/

├── config.py          # 全局常量与颜色定义
├── entities.py        # 游戏实体类 + 碰撞/边界检测
├── utils.py           # 食物生成、最高分 I/O、键盘输入
├── ui.py              # 界面渲染（HUD、菜单、暂停、结束画面）
├── .gitignore         # Git 忽略规则
├── README.md          # 项目说明文档
└── high_score.txt     # 最高分记录（运行时自动生成）
```
